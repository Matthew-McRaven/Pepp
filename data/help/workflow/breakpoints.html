Using Breakpoints

Applies to: Mc2, ISA3, Asmb3, OS4, Asmb5

Add / Remove Breakpoints

Click in the breakpoint gutter (the narrow strip to the left of code) next to an executable statement to toggle a breakpoint. A red marker indicates an active breakpoint. Click again to remove it.

In Asmb3, OS4, and Asmb5, breakpoints are synchronized between the Source and Listing panes. Setting (or clearing) a breakpoint in one pane updates the other automatically.

Run to a Breakpoint

To execute until the next breakpoint, use Debug → Continue (or the toolbar icon). While this example uses Continue, breakpoints can be hit under any step mode; see Step Modes.

When a Breakpoint is Hit

Execution pauses before the breakpointed instruction runs. The current statement is highlighted, and the surrounding debugging views (CPU dump, memory dump, stack trace, I/O panes) reflect the state at that point.

Managing Breakpoints

  • Clear All: Use Debug → Clear All Breakpoints to remove all breakpoints at once.

For bulk review and edits (enable/disable, navigate, delete), use the Breakpoint Viewer Advanced Topic.

Step Modes

Pepp provides four debugging step operations: Single Step, Step Into, Step Over, and Step Out. These control how the debugger executes CALL and SCALL instructions, and how it resumes after them. They can be used from the Debug menu or toolbar, and each has an associated keyboard shortcut.

Applies to: ISA3, Asmb3, OS4, Asmb5

Single Step

Executes one von Neumann cycle. If the current instruction is a CALL, acts as Step Into; otherwise acts as Step Over. This is the most common operation for line-by-line debugging.

Step Into

Executes the next instruction. If it is a CALL or SCALL, begins tracing inside the callee or OS trap handler. Useful when you want to examine internal behavior.

Step Over

Executes the next instruction. If it is a CALL or SCALL, runs it to completion as if it were a single instruction. Useful when you do not need to see the internal execution of the callee or OS trap.

Step Out

Resumes execution until the current CALL or SCALL returns to its caller. This is useful when you have stepped into a function or trap but now want to return to the higher level.

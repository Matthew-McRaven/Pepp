Step Modes

Applies to: ISA3, Asmb3, OS4, Asmb5

Pepp provides four debugging step operations: Single Step (1), Step Into (2), Step Over (3), and Step Out (4). These control how the debugger executes CALL and SCALL instructions, and how it resumes after them. They can be used from the Debug menu or toolbar, and each has an associated keyboard shortcut.

Single Step

Executes one von Neumann cycle. This is the most common operation for line-by-line debugging. Generally, this operation will execute a single line of code. If the current instruction is a CALL, Single Step acts as Step Into. Otherwise, Single Step acts as Step Over.

Step Into

Executes the next instruction. If it is a CALL or SCALL, begins tracing inside the callee or OS trap handler. Useful when you want to examine state changes in the stack, return value of the call, or internal behavior of the operating system call.

Step Over

Executes the next instruction. If it is a CALL or SCALL, runs it to completion as if it were a single instruction. Useful when you do not need to see the internal execution of the callee or OS trap.

Step Out

Resumes execution until the current CALL or SCALL returns to its caller. This is useful when you have stepped into a function or trap but now want to return to the higher level.

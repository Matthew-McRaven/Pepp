Machine Language (ISA3) Workflow

Applies to: ISA3

Running ISA3 Programs

1) Write your program
Enter hexadecimal bytes directly in the Object Code pane. Each byte is exactly two hex characters (0–9, A–F, a–f).

You may choose any number of spacing characters (including 0) to separate bytes, even with Pep/9. Our standard style uses one space between bytes, with 16 bytes per line, which is applied with automatic formatting.

While previous editions require a terminating zz as a loader sentinel, Pep/10 does not require this.

The Pep/10 Reference in this Help provides a list of opcodes and their bit patterns, which can be used to write programs in object code. This view is visible in the Editor mode by default. You can also use the byte converters to help convert between different data representations.

2) Load the program
Use Build → Load Object Code to load the program into memory. A load button also appears in the toolbar

3) Execute the program
Use Build → Execute to simulate the program or the execute button in the toolbar. Executing the program will switch you the Debugger mode if you are not already in it. Program input must be placed into the input pane before execution for memory-mapped IO to work correctly.

Program output appears in the Output pane.

Debugging ISA3 Programs

1) Start debugging
Use Debug → Start Debugging Object to clear memory to zero, load the program at 0000, and enter Debugger mode.

2) Initial state
At start, PC = 0000. The next instruction is highlighted in blue in the Memory Dump.

3) Step through execution
Use any Step Mode to advance. Blue = instruction to execute next; red = bytes written by the last step.

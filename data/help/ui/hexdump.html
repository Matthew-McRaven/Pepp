Memory Dump

Anatomy of a Memory Row

Each row has three parts, separated by vertical bars:

  • Left: The address (hex) of the first byte in the row.
  • Middle: Eight cells, each representing a single byte in the row.
  • Right: The ASCII rendering of the eight bytes, with non-printable characters shown as periods (.).

Hover over any byte cell to see a tooltip with the decoded value in hex, unsigned decimal, binary, and the opcode interpretation.

Highlights

  • Red: Bytes written since the last step. If a single step (including step over/out) executes multiple instructions, all bytes written during that step are highlighted.
  • Orange: The current value of SP.
  • Blue: The current value of PC.
Note: Exact colors may vary with the selected theme.

Navigation

  • Use the vertical scrollbar to move through memory.
  • Enter an address in the "Scroll to" field at the bottom to jump directly to a specific memory address.
  • Use the SP and PC buttons to jump to the current stack pointer and program counter, respectively.
